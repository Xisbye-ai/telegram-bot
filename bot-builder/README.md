# 🤖 Конструктор ботов

Программа для ПК, в которой ты **собираешь бота из блоков** — «найти картинку», «кликнуть»,
«если не нашёл — искать дальше» — и можешь **обучить нейросеть** находить объекты, которые
каждый раз выглядят по-разному (цвет, форма, положение). Интерфейс открывается в браузере,
поэтому **с телефона тоже видно**, как бот устроен и что он делает прямо сейчас.

Всё работает локально на твоём ПК, никуда в интернет ничего не отправляется.

## Установка и запуск

1. Установи [Python 3.10+](https://www.python.org/downloads/) (на Windows при установке
   отметь галочку **Add Python to PATH**).
2. **Windows:** просто запусти `start.bat` двойным кликом — при первом запуске он сам
   установит зависимости (скачается примерно 400 МБ, больше всего весит PyTorch для
   нейросети), а дальше сразу откроет программу в браузере.

   **Вручную (или на Linux/Mac):**

   ```
   pip install -r requirements.txt
   python run.py
   ```

В консоли появятся два адреса:

- **На этом ПК** — открой `http://localhost:8000` в браузере;
- **С телефона** — открой показанный адрес вида `http://192.168.x.x:8000`
  (телефон должен быть в том же Wi-Fi; если не открывается — разреши Python
  доступ к сети в брандмауэре Windows, окно с вопросом появляется при первом запуске).

С телефона видно всё то же самое: блоки сценария, журнал работы бота в реальном времени,
живой просмотр экрана ПК (вкладка «Экран» → «Живой просмотр»). Можно даже управлять:
запускать/останавливать бота и запускать обучение.

> ⚠️ Пароля нет, поэтому запускай только в домашней сети, которой доверяешь.
> Если доступ с телефона не нужен: `python run.py --host 127.0.0.1`

## Горячие клавиши

Работают глобально — даже когда открыта игра, а не браузер. Настройка — кнопка **⌨**
вверху страницы (клавиши F1–F12 или «выкл»):

| По умолчанию | Действие |
|---|---|
| **F9** | Запустить последний сценарий / остановить бота |
| **F10** | Аварийная остановка |
| **F8** | Снимок из игры → сразу открывается разметка для обучения нейросети |
| **F7** | Снимок из игры → выделение области или образца |
| (по желанию) | Начать / закончить запись действий |

То есть можно играть, увидеть камушки, нажать F8 — и на телефоне или втором мониторе
уже открыт свежий снимок: обводи рамками, жми «В обучение». Никакого переключения окон.

## Как собрать бота (вкладка «Редактор»)

Бот выполняет блоки сверху вниз. Кнопка «＋ Добавить блок» есть в конце списка и внутри
блоков-контейнеров. Нажми «✨ Пример», чтобы увидеть готовый сценарий.

- **«?» на каждом блоке** — подробное объяснение, как блок работает и как его настроить.
  Кнопка «❓ Справка по блокам» показывает все объяснения сразу.
- **Проверка логики.** Редактор сам находит недоделки: не выбран образец, «Клик по
  найденному» без блока поиска выше, «Для каждого» без «все совпадения», пустые ветки,
  бесконечный цикл без паузы… Ошибки подсвечиваются красным с объяснением, что
  исправить, предупреждения — жёлтым. Счётчик проблем виден рядом с кнопками сценария,
  при запуске с ошибками программа переспросит.
- **Отмена и удобство.** Ctrl+Z / Ctrl+Y (или кнопки ↶ ↷) отменяют и возвращают
  изменения. Блоки можно перетаскивать мышью за значок ⠿, а большие циклы —
  сворачивать (▾/▸).
- **⏺ Запись действий (макрорекордер).** Нажми «Записать действия», поиграй —
  программа сама превратит твои клики, нажатия клавиш, набор текста и прокрутку в блоки
  и добавит их в конец сценария. Дальше правь как обычно. Двойные клики и слова
  распознаются автоматически, паузы между действиями сохраняются. Можно повесить на
  горячую клавишу — тогда старт/стоп прямо из игры. (Нужен `pynput`.)
- **Привязка к окну игры.** Поле «Окно игры» (часть заголовка) заставляет бота искать и
  кликать только внутри окна этой игры, даже если окно передвинули или оно не на весь
  экран. Кнопка 🪟 обновляет список открытых окон. Пусто — работа по всему экрану.
  (Нужен `pygetwindow`, Windows.)

### Видно, что делает бот («глаза бота»)

Пока бот работает: блок, который выполняется прямо сейчас, подсвечивается зелёным в
редакторе (список сам прокручивается к нему), а на вкладке «Экран» при включённом
«Живом просмотре» прямо на картинке рисуются рамки вокруг того, что бот нашёл, и круги
там, куда он кликнул. Смотри с телефона — и видишь не сухой журнал, а глазами бота.

| Блок | Что делает |
|---|---|
| 🔍 Найти картинку | Ищет на экране сохранённый кусочек экрана (образец). Режим «Ждать, пока появится» — ищет снова и снова, пока не найдёт. Можно искать только в сохранённой области экрана |
| 🧠 Найти объект (нейросеть) | То же, но ищет обученная нейросеть — для объектов, которые выглядят по-разному |
| 🔤 Прочитать текст (OCR) | Читает текст или число из области экрана (количество золота, HP…) и кладёт в счётчик |
| 🖱 Клик | Кликает по тому, что нашлось, или по координатам. «Разброс точки» — кликать каждый раз чуть по-разному |
| ✊ Перетащить мышью | Зажать, перетащить, отпустить — для инвентаря |
| ⌨️ Удерживать клавишу | Держит клавишу N секунд (бег, полёт) |
| ⌨️ Ввести текст / Нажать клавишу | Печатает текст, жмёт клавиши (`enter`, `esc`, `ctrl+c`…) |
| ⏳ Пауза, 🖲 Прокрутка | Ждёт (можно «± случайно» — по-человечески), крутит колесо мыши |
| 🔀 Если найдено… | Две ветки: что делать, если нашлось, и что — если нет |
| 📍 Для каждого найденного | Если в поиске выбрано «Все совпадения» — выполняет блоки для каждой находки по очереди (например, кликнуть по каждому камушку) |
| 🎨 Если цвет пикселя | Проверяет цвет точки экрана — например, «полоска HP ещё красная?». Координаты и цвет подскажет вкладка «Экран»: короткое нажатие по снимку |
| 🔢 Счётчик, 🧮 Если счётчик… | Считает события и сравнивает: «пока не соберу 10 штук» |
| 🔁 Повторить N раз, ♾ Повторять бесконечно | Циклы |
| 💬 Сообщение, ⏹ Стоп | Пишет в журнал, останавливает бота |
| 🖥 Показать на HUD / 🧹 Очистить HUD | Строки поверх игры и в браузере: «Собрано: {камни}» |
| 📊 Записать статистику | Строка в таблицу на вкладке «Статистика» (и в CSV-файл) |
| 🔔 Сигнал | Звук и уведомление в открытых вкладках — в том числе на телефоне |

В текстах блоков работают подстановки: `{имя_счётчика}`, `{найдено}` (сколько нашлось),
`{текст}` (последний прочитанный текст), `{время}`.

Типичный сценарий «искать, пока не найдёшь, потом кликнуть»:

```
♾ Повторять бесконечно
   🔍 Найти картинку (режим: ждать, пока появится)
   🔀 Если найдено
      ✅ → 🖱 Клик по найденному → ⏳ Пауза 2 сек
```

Сценарий «собрать все камушки на экране, посчитать и показать на HUD»:

```
♾ Повторять бесконечно
   🧠 Найти объект (что искать: все совпадения)
   📍 Для каждого найденного
      🖱 Клик по найденному (разброс точки: 3)
      🔢 Счётчик «камни» +1
      ⏳ Пауза 0.5 ± 0.3 сек
   🖥 Показать на HUD: «Собрано камней: {камни}»
   🧮 Если счётчик «камни» ≥ 50
      ✅ → 📊 Записать статистику → 🔔 Сигнал → ⏹ Стоп
```

Кнопки «⬇ Файл» и «⬆ Из файла» скачивают и загружают сценарий как обычный файл —
удобно для резервных копий и обмена.

## HUD поверх игры

Блок «🖥 Показать на HUD» выводит строки в полупрозрачное окошко в правом верхнем
углу экрана — поверх игры, клики сквозь него проходят. Те же строки видны зелёной
панелью в браузере (значит, и на телефоне). Собери свой HUD блоками: какие строки,
что в них писать и когда обновлять.

## Чтение текста (OCR)

Блок «🔤 Прочитать текст» умеет читать надписи и числа с экрана (золото, HP, уровень):

1. Установи программу Tesseract (один раз):
   **Windows** — установщик <https://github.com/UB-Mannheim/tesseract/wiki>
   (в списке компонентов отметь Russian, если нужен русский текст);
   **Linux** — `sudo apt install tesseract-ocr tesseract-ocr-rus`.
2. На вкладке «Экран» в режиме «Область» выдели место с текстом, кнопка
   «🔤 Прочитать текст» сразу покажет, что распозналось. Сохрани область с именем.
3. В сценарии блок «Прочитать текст» кладёт результат в счётчик — дальше его можно
   сравнивать («🧮 Если счётчик…»), показывать на HUD и писать в статистику.

Для чисел включай «Только числа» — так распознавание заметно надёжнее.

Картинки-образцы делаются на вкладке **«Экран и образцы»**: сделай снимок, выдели
нужное место мышью, дай имя, сохрани. Кнопка «Через 5 сек» даёт время переключиться
на окно игры.

## Как обучить нейросеть (вкладка «Обучение»)

Для объектов, которые каждый раз разные (например, камушки в игре):

1. **Собери примеры.** Вкладка «Экран» → сделай снимок → режим «Разметка» → обведи
   каждый камушек рамкой (класс, например, `камушек`) → «📚 В обучение».
   Повтори на 5–10 разных снимках, чтобы набралось 30–50 рамок. Чем разнообразнее
   примеры (разный фон, цвет, размер), тем лучше нейросеть поймёт, что такое «камушек».

   Быстрее набрать кадры помогает **⏱ Автосъёмка** (на вкладке «Экран»): программа сама
   делает снимки каждые N секунд, пока ты играешь, — потом останется только обвести
   объекты на вкладке «Обучение» (кнопка «✏ Рамки»). А галочка **🧪 «Собирать сложные
   кадры»** сохраняет на доразметку те моменты, когда уже обученная нейросеть ничего не
   нашла, — так модель растёт на собственных ошибках.
2. **Обучи.** Вкладка «Обучение» → имя модели → «🧠 Обучить». На обычном ПК это
   несколько минут; ход виден по шкале и в журнале.
3. **Проверь.** Кнопка «🔎 Найти на экране сейчас» покажет, что модель находит.
   Плохо находит — добавь ещё снимков с рамками и обучи заново. Находит лишнее —
   подними «уверенность».
4. **Используй.** В редакторе добавь блок «🧠 Найти объект (нейросеть)» и выбери модель.

В одну модель можно разметить несколько классов (камушки, монетки…) — в блоке поиска
потом выбирается, что именно искать.

## Полезно знать

- **Аварийный стоп:** нажми **F10** (настраивается) или резко дёрни мышь в левый верхний
  угол экрана — бот сразу остановится. Кнопка «⏹ Стоп» тоже работает (в том числе с телефона).
- Бот ищет и кликает на **основном мониторе**.
- Образец ищется «как есть», поэтому если поменять масштаб/разрешение игры — сохрани
  образец заново. Нейросеть к таким изменениям устойчивее.
- Нейропоиск заметно медленнее поиска по картинке (секунды на проверку) — это нормально.
- Все твои данные лежат в папке `bot-builder/data`: сценарии, образцы, снимки для
  обучения и модели. Можно копировать как резервную копию.

## Как это устроено (для любопытных)

- `run.py` — запуск локального веб-сервера (FastAPI + uvicorn).
- `app/main.py` — API: сценарии, снимки экрана, образцы, области, датасет, обучение,
  статистика, журнал и HUD (WebSocket).
- `app/engine.py` — исполнитель блоков: поток, который снимает экран (`mss`), ищет
  (`OpenCV` или нейросеть) и управляет мышью/клавиатурой (`pyautogui`).
- `app/detector.py` — нейросеть на PyTorch: маленькая свёрточная сеть учится отличать
  кусочки экрана 64×64 («фон или объект»), а потом за один проход строит карту
  вероятностей по всему экрану в нескольких масштабах.
- `app/ocr.py` — чтение текста через Tesseract.
- `app/overlay.py` — HUD-окошко поверх игры (tkinter, всегда сверху, клики проходят сквозь).
- `app/recorder.py` — запись действий (pynput) и превращение их в блоки сценария.
- `app/static/` — весь интерфейс: HTML + CSS + JavaScript без сторонних библиотек.
